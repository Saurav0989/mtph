The <mtph-doc> web component

Drop one <script> tag on any page, and a .mtph problem renders live — real LaTeX, compiled figures, self-check quizzes, even sliders — with nothing to install and no build step for the page.

<script src="mtph-doc.js"></script>

<mtph-doc>
  <script type="text/plain">
  ---
  mtph: "0.2"
  title: My problem
  ---
  A block of mass $m$ on an incline of angle $\theta$ ...
  </script>
</mtph-doc>

The problem renders inside a sandboxed iframe fed the exact self-contained page the CLI produces, so its styling and KaTeX never leak into (or out of) your page. The frame reports its height back, so it grows to fit — no inner scrollbar.

1. A problem, rendered

Inline source via a <script type="text/plain"> child. The answer & solution sit in a collapsed reveal.

2. Quiz mode quiz

Add the quiz attribute and a numeric answer becomes an input graded against its tolerance; a multiple-choice answer becomes clickable buttons.

3. Explorable — sliders from params:

Declare params: and reference them as {{name}} in a figure or plot. Sliders appear above the problem and it re-renders live as you drag. (The stored default stays authoritative, so static/PNG output is still deterministic.)

4. Statement only hide-answers

Add hide-answers to drop the answer and solution entirely — handy for embedding a problem set students shouldn't see the key for.

5. From a file src="…"

Point it at a .mtph URL instead of inlining. (This one loads a sibling file, so it needs the page served over HTTP — locally, run any static server in playground/.)

Part of mtph — an AI-native file format for math & physics problems. The same renderer powers the playground and Claude Artifacts.
